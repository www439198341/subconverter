## 命令
```
docker ps
# 获取运行中的container_id

docker exec -it ${container_id} /bin/sh
# 进入实例shell交互环境

vi rules/LocalAreaNetwork.list
# 增加需要的规则. 规则模板参照后续

exit
# 退出交互

docker commit -m "${description}" -a "${username}" ${container_id} ${image_name}:${tag}
# example:
# description 可以是任意字符串
# username 可以是任意名字
# image_name 可以是任意名字
# tag 可以是任意标签名, 比如v1, latest等

docker stop ${container_id}
# 停止正在运行的实例

 docker run -d --restart=always -p 25500:25500 ${image_name}:${tag}
 # 启动最新的image实例
```

## 规则模板
```
IP-CIDR,xx.xx.xx.xx,no-resolve
# 该ip地址走直连策略

DOMAIN,xxx.com,no-resolve
# 该域名走直连策略


```

## 更新image
```
docker image ls
# 查看所有的image

docker image rm ${image_id}
# 删除image

docker ps -a
# 查看所有运行的容器(包含已停止的)

docker rm ${container_id}
# 删除容器(删除已停止的)

docker stop ${container_id}
# 停止运行中的容器

docker pull tindy2013/subconverter
# 从hub.docker拉取最新版本

# 重新添加自定义规则并启动新的容器
```
